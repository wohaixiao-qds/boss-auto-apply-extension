# BOSS 自动打招呼插件使用说明

## 项目简介

这是一个基于 Chrome Side Panel 的 BOSS 直聘自动打招呼扩展。插件在浏览器侧边栏运行，不覆盖或改变 BOSS 页面展示。

插件只读取 BOSS 页面和页面请求中的职位信息，发送时使用 BOSS 原生页面交互，不调用未公开的发送接口，也不绕过验证码、登录校验和频率限制。

## 技术栈

- TypeScript
- React
- WXT + Vite
- Radix Themes
- Lucide React
- Chrome Manifest V3
- Chrome Storage API
- Node.js 20 LTS

## 本地开发

```bash
npm install
npm run dev
```

生产构建：

```bash
npm run build
```

生成可分发压缩包：

```bash
npm run package
```

压缩包输出到 `release/boss-auto-greeting.zip`。

## 在 Chrome 中安装

1. 执行 `npm run build`。
2. 打开 Chrome 的 `chrome://extensions/`。
3. 开启右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择项目下的 `.output/chrome-mv3` 文件夹。
6. 打开 BOSS 直聘并确认已经登录。
7. 点击 Chrome 工具栏中的扩展图标，打开侧边栏。

如果使用压缩包，先解压 `release/boss-auto-greeting.zip`，再在 Chrome 中选择解压后的文件夹。

## 重要说明：这是半自动插件

本插件不是打开后自动遍历全网职位的全自动工具。用户需要先在 BOSS 直聘中完成职位筛选并确认投递范围，插件再根据扫描结果执行批量打招呼。

具体分工如下：

- 用户：登录 BOSS、设置筛选条件、扫描职位、审核职位、确认开始。
- 插件：读取当前筛选结果，过滤非岗位入口和外包岗位，按职位 ID 建立队列，并按列表顺序逐个执行打招呼。
- 用户可随时：暂停、继续、停止任务，移除待投递职位，切换查看成功或失败列表。

插件不会自动修改 BOSS 筛选条件，也不会自动处理验证码、登录失效或频率限制。遇到这些情况时会暂停任务，需要用户在 BOSS 页面处理后再继续。

## 使用流程

1. 登录 BOSS 直聘，并进入职位列表页。
2. 在 BOSS 职位列表页手动完成城市、关键词、薪资、经验等筛选。
3. 保持 BOSS 职位列表页打开，点击 Chrome 工具栏中的扩展图标打开侧边栏。
4. 点击“扫描当前 BOSS 列表”，等待职位列表读取完成。
5. 在侧边栏审核扫描结果，可移除不需要投递的职位。
6. 在“投递设置”中配置：
   - 是否排除外包岗位；
   - 每批最多投递数量；
7. 点击“同意并按顺序发送”，在确认弹窗中点击“同意开始”。
8. 任务执行期间不要刷新 BOSS 页面或手动切换职位页面；可以在侧边栏暂停、继续或停止任务。
9. 插件会在后台逐个打开职位详情页并使用 BOSS 原生页面交互发送，处理完成后关闭详情页。
10. 任务完成后，侧边栏会显示本批次结果，并尝试发送 Chrome 系统通知。系统会自动将职位分到“投递成功”和“投递失败”列表。

系统会在职位之间使用随机等待时间，以降低连续操作对页面的影响；该等待时间由插件自动管理，不需要用户单独配置。

## 更新插件后

如果是通过“加载已解压的扩展程序”安装：

1. 拉取最新代码并执行 `npm run build`。
2. 打开 `chrome://extensions/`。
3. 点击插件卡片上的“重新加载”。
4. 重新打开侧边栏即可使用最新版本。

## 职位状态

- 待投递：已扫描但尚未处理。
- 投递中：当前正在处理。
- 投递成功：已检测到 BOSS 原生页面的成功结果。
- 投递失败：页面异常、按钮不可用、登录失效或其他原因导致未完成。
- 已跳过：职位不满足投递条件或被过滤。

职位使用页面读取到的 `jobId` 关联，避免仅依赖公司名或职位名称造成误匹配。

## 注意事项

- 使用前请保持 BOSS 账号处于登录状态。
- 请不要在任务执行期间频繁刷新或手动切换 BOSS 页面。
- 遇到验证码、频率限制、登录失效或页面结构异常时，任务会自动暂停，请根据页面提示处理。
- 插件数据仅保存在浏览器本地，不连接外部 AI 或后端服务。
- Chrome 系统通知受操作系统通知权限、勿扰模式和浏览器设置影响。
